# Tailwind CSS + SASS Integration Guide

## Overview
This project combines **Tailwind CSS** (utility-first CSS framework) with **SASS** (preprocessor with variables, mixins, nesting) to achieve a balance between speed and abstraction.

## Why Use Both?

### Tailwind CSS
Tailwind provides:
- **Utility classes** for layout (grid, flex), spacing, and responsive behavior
- **Rapid prototyping** — apply styles directly in markup without writing CSS
- **Consistency** — predefined design tokens (colors, spacing, typography)
- **Minimal CSS output** — only used utilities are included in the final build

**Used for:** Layout, responsive design, spacing, typography, quick styling

### SASS
SASS provides:
- **Design tokens** — centralized, reusable variables (colors, spacing, fonts)
- **DRY abstraction** — mixins, functions, placeholders to reduce repetition
- **Complex selectors** — BEM nesting, state variants, component abstractions
- **Maintainability** — easier to refactor and extend component patterns

**Used for:** Component styles, design tokens, complex abstractions, state variants

## Build Pipeline

```
SASS Compilation → Compiled CSS → PostCSS (Tailwind) → Final CSS
```

### Step-by-step
1. **SASS compiles** (`src/styles/main.scss`) to plain CSS (`src/styles/main.css`)
2. **PostCSS/Tailwind processes**:
   - Injects Tailwind's base, component, and utility classes
   - Resolves `@apply` directives (e.g., `@apply text-lg font-bold`)
   - Minifies and optimizes the output
3. **Final output** → `public/tailwind.css`

## File Structure

```
src/styles/
├── main.scss                 # Entry file (loads Tailwind + all partials)
├── _variables.scss           # Design tokens (colors, spacing, fonts, shadows)
├── _mixins.scss              # Reusable mixins (flex-center, breakpoints, transitions)
├── _placeholders.scss        # Placeholder classes (%card-base, %button-base, etc.)
├── _layout.scss              # Layout patterns (grid, sidebar layouts, sections)
└── _components.scss          # Component styles (card, button, input, badge, ticker)

public/
└── tailwind.css              # Compiled final CSS (generated by build)
```

## How They Work Together

### Example: Card Component

**HTML (markup uses Tailwind classes + component class)**
```html
<div class="card card--highlight">
  <div class="card__header">
    <h3 class="card__title">Title</h3>
    <p class="card__subtitle">Subtitle</p>
  </div>
  <div class="card__body">
    <p>Content here</p>
  </div>
  <div class="card__footer">
    <button class="btn btn--secondary">Cancel</button>
    <button class="btn">Confirm</button>
  </div>
</div>
```

**SASS (_components.scss)**
```scss
.card {
  @extend %card-base;           // Placeholder: base card styles (shadow, radius, hover)
  padding: $spacing-lg;          // Variable: 1.5rem

  &__title {
    @apply text-xl font-semibold; // Tailwind utilities applied via @apply
    color: $color-gray-900;      // Variable: #111827
  }

  &--highlight {
    border-left: 4px solid $brand-primary; // Variable: #0ea5a4
  }
}
```

### Rationale
- **`@extend %card-base`** — Avoids repeating shadow, radius, hover state
- **`@apply text-xl font-semibold`** — Reuses Tailwind utilities inside component class
- **`$color-gray-900`** — Single source of truth for colors across the site
- **`$spacing-lg`** — Consistent spacing scale

## Build Commands

### Production Build
```bash
npm run build:css
```
Compiles SASS → processes Tailwind → minified output to `public/tailwind.css`

### Development (Watch Mode)
```bash
npm run dev:css
```
Watches SASS files and rebuilds CSS automatically (two concurrent watchers)

### Full Project Build
```bash
npm run build
```
Builds both JS and CSS together

## Design Tokens Reference

### Colors
- **Brand:** `$brand-primary` (#0ea5a4), `$brand-secondary` (#1f2937)
- **Status:** `$color-success`, `$color-error`, `$color-warning`, `$color-info`
- **Grayscale:** `$color-gray-50` through `$color-gray-900`

### Spacing (rem-based)
`$spacing-xs` (0.25rem) → `$spacing-3xl` (4rem)

### Border Radius
`$radius-sm` (0.25rem) → `$radius-full` (9999px)

### Typography
- **Fonts:** `$font-family-sans` (Inter), `$font-family-serif` (Playfair Display)
- **Sizes:** `$font-size-xs` (0.75rem) → `$font-size-4xl` (2.25rem)
- **Weights:** `$font-weight-light` (300) → `$font-weight-extrabold` (800)

### Shadows
`$shadow-sm` → `$shadow-xl`, plus `$shadow-card` for default card styling

## Mixins

**Flexbox:**
- `@include flex-center;` — Centers items horizontally and vertically
- `@include flex-between;` — Spaces items apart

**Breakpoints (mobile-first):**
- `@include sm { }` — 640px+
- `@include md { }` — 768px+
- `@include lg { }` — 1024px+
- `@include xl { }` — 1280px+
- `@include breakpoint-2xl { }` — 1536px+

**Utilities:**
- `@include truncate;` — Ellipsis on overflow text
- `@include line-clamp($lines);` — Limit lines with ellipsis
- `@include focus-ring;` — Accessible focus outline
- `@include transition($property);` — Smooth transitions
- `@include absolute-center;` — Center absolute-positioned elements
- `@include button-reset;` — Remove browser button styles

## Placeholders

Placeholders (`%name`) are abstract base styles you extend:
- `%card-base` — Foundation for all cards
- `%button-base` — Foundation for all buttons
- `%input-base` — Foundation for all inputs
- `%section-padding` — Consistent section spacing
- `%container` — Centered max-width wrapper
- `%sr-only` — Visually hidden (screen-reader only)

**Usage:**
```scss
.my-button {
  @extend %button-base;
  // Additional styles
}
```

## Migrating Existing CSS

**Gradual approach:**
1. Keep old CSS files (`styles/index.css`, `styles/slider.css`) temporarily
2. Add new component styles in `src/styles/_components.scss`
3. Convert old components one at a time:
   - Move hardcoded values to `_variables.scss`
   - Replace repeated rules with mixins or placeholders
   - Use `@apply` to reference Tailwind utilities
4. Remove old CSS once fully migrated

**In HTML, you can gradually replace:**
```html
<!-- Old -->
<div class="old-card">...</div>

<!-- New -->
<div class="card">...</div>
```

## Best Practices

1. **Use Tailwind for:**
   - Layout (grid, flexbox)
   - One-off spacing/sizing
   - Responsive breakpoints
   - Typography utilities in markup

2. **Use SASS for:**
   - Design tokens (variables)
   - Repeated patterns (mixins, placeholders)
   - Component abstractions (nesting, BEM)
   - Complex state variants

3. **Use `@apply` when:**
   - A component uses many Tailwind utilities together
   - You want to reuse a utility group inside a class

4. **Keep it DRY:**
   - Extract magic numbers to variables
   - Create mixins for repeated patterns
   - Use placeholders for base styles

## Troubleshooting

### CSS not updating?
- Make sure `npm run dev:css` is running in a terminal
- Check `public/tailwind.css` was generated (should be >10KB)
- Verify `<link rel="stylesheet" href="public/tailwind.css" />` is in `index.html`

### Tailwind utilities not appearing?
- Ensure `@tailwind utilities;` is at the end of `main.scss`
- Check `tailwind.config.cjs` includes your HTML file paths in `content:`

### SASS variables not accessible in components?
- Add `@use 'variables' as *;` at the top of the file that uses them
- SASS uses module system; namespaced imports require prefixes (unless using `as *`)
